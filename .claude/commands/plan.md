根據目標與當前體能狀態，規劃並建立訓練課表到 intervals.icu。

執行前會先展示課表讓你確認，確認後才寫入。

## 參數說明

`$ARGUMENTS` 可以是：
- 空白 → 規劃下週課表
- `2` → 規劃未來 2 週
- `2026-08` → 規劃指定月份

## 步驟

### 第一步：讀取所有脈絡

讀取 `goals.md`：目標賽事、日期、FTP 目標、CTL 目標、里程碑。
讀取 `coach/review_rules.md`：訓練原則與強度規範。

### 第二步：讀取當前狀態

讀取最近 wellness 資料（最近 14 天的 `data/YYYY/MM/wellness.json`），取得最新 CTL、ATL、TSB。
讀取最近 14 天活動，了解目前訓練頻率與強度習慣。

### 第三步：查詢現有排程

使用 `get_upcoming_events` 工具查詢 intervals.icu 上已有的課表，避免衝突。

### 第四步：規劃課表

依以下原則規劃：
- 週 TSS 增幅不超過 10%（CTL 目標未達前）
- 每 3 週漸進 + 1 週恢復
- TSB 目前 < -20 時，優先安排恢復
- 高強度課（Z4/Z5）後需有恢復日
- 目標賽事前 2 週進行 taper（週 TSS 減 30-40%）
- 每週安排符合 `goals.md` 目標的重點訓練

### 第五步：展示課表，等待確認

以清單展示每天的課程安排，格式如下：

```
📋 課表草案（2026-07-07 ~ 2026-07-13）

週一 07/07  休息
週二 07/08  Z2 有氧 90 分鐘（TSS ~65）
週三 07/09  閾值間歇 2×20min（TSS ~85）
週四 07/10  恢復騎 45 分鐘（TSS ~25）
週五 07/11  休息
週六 07/12  長騎 Z2 3 小時（TSS ~120）
週日 07/13  Z2 有氧 60 分鐘（TSS ~45）

本週預計 TSS：340（目前週 TSS 約 XXX）
預計期末 CTL：XX → XX

確認建立到 intervals.icu？（輸入 yes 確認）
```

### 第六步：確認後建立

使用者輸入 `yes` 後，使用 `create_workout` 工具逐筆建立課表。
每建立一筆印出確認訊息。
全部完成後，告知使用者可至 intervals.icu 行事曆查看。

用繁體中文回覆，技術術語保留英文縮寫。
